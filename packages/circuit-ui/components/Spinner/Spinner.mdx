## Accessibility

> This section is a stub. Contributions welcome!

- The Spinner component doesn't accept a `label` prop. If your UI loads in a significant amount of time, make sure to add alternative text for users with vision deficiencies, for example using `aria-label`.
- Rendering a Spinner is not sufficient to make a loading state accessible. Set [`aria-busy`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes/aria-busy) on the loading area and decide whether it makes sense to announce any updated content using a [live region](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/ARIA_Live_Regions).
- The Spinner's animation can be considered essential when a UI is loading, therefore the pattern doesn't fail [2.2.2 Pause Stop Hide](https://www.w3.org/WAI/WCAG21/Understanding/pause-stop-hide.html)
